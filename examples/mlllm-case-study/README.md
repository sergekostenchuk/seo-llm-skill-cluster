# mlllm.io Case Study Outputs

This directory contains example artifacts produced during a dry-run of the SEO/LLM skill cluster against the mlllm.io-style website model.

The goal is to show what the skills produce, not to claim that every possible live SEO or LLM signal has been measured.

## Files

| File | What it demonstrates |
| --- | --- |
| `goal-brief.md` | Site objective, constraints, and success criteria |
| `routing-and-handoffs.md` | How the orchestrator routes work across specialist skills |
| `semantic-core.md` | Topics, intents, entities, and audience framing |
| `url-map.md` | Public URL structure and page roles |
| `internal-link-graph.md` | Brief, longform, topic, project, and author link model |
| `technical-seo-schema-audit.md` | Metadata, schema.org, hreflang, sitemap, RSS, and discovery checks |
| `llm-friendly-audit.md` | LLM-readable architecture, `llms.txt`, source trails, and agent traversal notes |
| `improvement-backlog.md` | Follow-up work, grouped by risk and priority |
| `homepage-seo-report.json` | Machine-readable homepage regression result |
| `en-news-seo-report.json` | Machine-readable short-news regression result |
| `en-article-seo-report.json` | Machine-readable longform regression result |
| `site-access-snapshot.json` | Public discovery and access snapshot |

## Important Boundary

These files are example outputs. They are useful for understanding the workflow and expected artifact shape, but they are not a live ranking guarantee, assistant-citation guarantee, or third-party SEO certification.
